UNIFY Identity Broker for Microsoft FIM v4.0.0 Configuration Guide

 

Installation Notes

·         Run the MSI.

·         The installer for the adapter defaults to the Services/Microsoft.FIM directory, and contains a Unify.IdentityBroker.FIMAdapter.dll file.

·         For a Microsoft Forefront Identity Manager instance, this file must be placed in the Microsoft Forefront Identity Manager/2010/Synchronization Service/Extensions directory. The miisserver.exe.config file in the bin directory of FIM must also be configured with the correct WCF endpoint information. Here is an example of what the file should look like:

<?xml version="1.0" encoding="UTF-16"?>

<configuration>

<startup>

<requiredRuntime version="v2.0.50727">

</requiredRuntime>

<supportedRuntime version="v2.0.50727">

</supportedRuntime>

</startup>

<system.serviceModel>
<bindings>
<basicHttpBinding>
<binding name="MetadataExchangeHttpBinding_ILDIFAdapter" closeTimeout="10:00:00"
openTimeout="10:00:00" receiveTimeout="10:00:00" sendTimeout="10:00:00"
maxBufferSize="65536" maxReceivedMessageSize="204003200" transferMode="Streamed" />
</basicHttpBinding>
</bindings>
<client>
<endpoint address=""
binding="basicHttpBinding" bindingConfiguration="MetadataExchangeHttpBinding_ILDIFAdapter"
contract="IdentityBrokerLDIFAdapter.ILDIFAdapter" name="ILDIFAdapter" />

</client>
</system.serviceModel>

 

</configuration>

 

·         After this, the FIM service must be restarted.
